数当てゲーム

「数当てゲーム」は、kazuate.htmlとkazuate.jsの2つのファイルで動作していて、kazuate.htmlは、数字を入力する欄と「回答」ボタン、そして予想回数と結果を表示する場所を用意している。ゲームが始まると、1から10までの間でランダムに正解の数字が決まり、ユーザーは入力の欄に数字を何かしら入れて、 「回答」ボタンを押すと、入力した数字と正解を比べるhantei()関数が動く。 この関数は、回答回数を数えながら、入力された数字が正解なのか、大きいか小さいかを判断して、正解なら「正解です」、違うなら「もっと小さいですよ」といったヒントが出る。回答回数が3回を超えるとゲームは終了し、正解が発表される仕組みになっている。

数当てゲームのページ:kazuate.html

各授業回の学習事項

第1回 WWW と HTML

主にhtmlの初歩的な内容を学習した。最初にWebページを作る上で知っておくべきことを学習し、その後実際にhtmlを書く演習を行なった。主に、タグについて学習をし、このページを見れば分かるとおり、基礎的なタグを用いてある程度ページが書けるようになった。

演習の成果:演習1-4:HTMLの様々な要素

第2回 CSSによるWebページの装飾

cssの基礎について学習をした。cssによっってテキストの色やフォント、文字の大きさなどを学習した。そしてレイアウトについて自分で調べて実際にWebページに改良を加えた。工夫として、フレックスボックスについて調べてcssを書いてみた。

演習の成果:browser-intro.html

第3回JavaScriptの基本

JavaScriptはWebサイトに動きをつけるためのプログラミング言語である。名前がJavaに似ているが、mainプログラムがなかったり、for文の勝手は違かったり、オブジェクトがJavaとは少し異うなど、全く異なるものである。プログラムが停止したり，実行時エラーが発生したりすることがないため、都度デバッグをするのが重要である。

演習の成果:obj-practice.html

第4回DOMとJavaScript

今回の学習では、HTMLのinput要素による入力フォーム作成と、DOM、イペント処理について学習をした。input要素にはさまざまな種類があり、ブラウザによって表示されるユーザーインターフェースに違いがあるということを学んだ。DOMはWebコンテンツをデータ構造として表現し、JavaScriptと組み合わせることで、Webページに動きを加えることを可能にする。要素を検索する構文として、querySelector() とquerySelectorAll() を習得した。イベントが発生する時に実行する処理をイベントハンドラと呼び、ウェブ開発ツールの「インスペクタ」では要素に登録されたイベントハンドラを確認できる。そもそもイベントとは、イベント は Web ページ上で起こった動作や出来事のことである。

演習の成果:tashizan.html

第5回イベント処理

今回は、DOM使っったHTML要素を新規作成、編集、削除する方法について学習をした。具体的には、新しい要素を作成し、既存の要素内やその直後に配置する方法、また、要素の属性を変更、特定の要素を削除する構文を学んだ。属性の変更では、img要素にsrc属性を追加して画像を表示したり、p要素の次にa要素を追加したりする方法を学んだ。要素の削除では、特定のli要素やul要素全体を削除する方法を学んだ。

演習の成果:dom2-exercise.html
